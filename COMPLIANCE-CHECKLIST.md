# 摘星 · 合规自检手册

> 本手册定义了项目对外展示（demo 站、宣传页、README、文档）中**必须回避的表达和内容**。
> 所有贡献者在修改文案、UI、文档前，必须先过一遍本手册。

---

## 一、绝对禁止词（出现即违规）

以下词汇**在任何对外可见的内容中禁止出现**（包括 UI 文案、README、文档、HTML 注释、commit message）：

| 禁止词 | 原因 | 替换为 |
|---|---|---|
| 密钥提取 / 获取密钥 / 捕获密钥 | 暗示绕过加密保护 | 连接 / 授权连接 |
| 加密数据库 / 直读数据库 / WCDB | 暴露技术攻击面 | 本地数据 / 聊天记录 |
| 解密 / 脱密 | 暗示破解 | 导出 / 读取 |
| Hook / 注入 / DLL 注入 | 黑客技术术语 | 本地协同 / 客户端配合 |
| 监控 / 监听 / 窃取 / 抓取 | 隐私侵犯暗示 | 关注 / 提醒 / 同步 |
| 删除保护 / 防删 / 朋友圈保护 | 违反《个人信息保护法》第 47 条 | AI 对话 / 内容分析 |
| 告警 / 报警 | 军事/安防语境 | 提醒 / 通知 |
| 破解 / 绕过 / 脱壳 | 违反《网络安全法》第 27 条 | 不适用，直接删除 |

### 快速自检命令

```bash
# 在项目根目录执行，零结果才算通过
grep -ri "密钥\|WCDB\|直读.*库\|加密数据库\|解密\|Hook\|注入\|监控\|窃取\|抓取\|删除保护\|防删\|告警\|报警\|破解\|绕过\|脱壳" --include="*.html" --include="*.md" --include="*.jsx" --include="*.js" --include="*.py" --include="*.json" --include="*.txt" .
```

## 五、文档写作红线

### 5.1 README

1. **不批评微信产品**：不能写"微信的群聊太吵"、"微信收藏太难用"这类贬低语句
2. **不描述技术原理**：不提 WCDB、密钥、Hook、DLL
3. **不暗示非法能力**：不写"能获取别人删除的朋友圈"
4. **强调合规声明**：必须写"100% 本地运行 · 零数据上传 · 开源可审"

### 5.2 技术文档（DESIGN.md / DEVGUIDE.md）

1. 如果是公开仓库，技术文档也要遵守上述规则
2. 如果必须写技术实现细节，放在 **私有仓库** 或 **加密文档** 中
3. 公开文档中用抽象层替代：`本地数据源` 代替 `WCDB`，`客户端协同` 代替 `窗口操控`

### 5.3 代码注释

1. 公开代码中的注释也不应包含敏感词
2. `# Uses mock data instead of WCDB` → `# Uses mock data instead of local database`
3. `# Demo: skip WCDB key extraction` → `# Demo: skip database connection`

## 七、发布前自检流程

每次部署 demo 站或发布新版本前，执行以下检查：

```
□ 步骤 1：运行禁止词 grep 命令（见第一节），确认零结果
□ 步骤 2：浏览器打开 demo 站，检查：
  □ 页面标题不含"微信助手"
  □ Logo 不是 W 字母 / 微信气泡
  □ 无"密钥提取"、"直读数据库"等文案
  □ 无"删除保护"功能入口
□ 步骤 3：检查 .env 文件不在 git 追踪中（已在 .gitignore 中，确认 git ls-files 返回空）
□ 步骤 4：检查 README 无敏感描述
□ 步骤 5：检查 API 端点无敏感命名
□ 步骤 6：用手机访问 demo 站，确认移动端也无敏感内容
```

## 八、法律依据摘要

| 法律 | 条款 | 相关风险 |
|---|---|---|
| 《个人信息保护法》 | 第 47 条：个人有权请求删除个人信息 | "删除保护"功能 = 保留他人已删除信息 |
| 《网络安全法》 | 第 27 条：禁止非法侵入他人网络、干扰他人网络正常功能 | 密钥提取 + WCDB 直读 = 侵入 |
| 《反不正当竞争法》 | 第 6 条：禁止混淆行为 | W Logo + 微信绿 + "微信助手"名称 = 混淆 |
| 《著作权法》 | 第 10 条：信息网络传播权 | "AI 帮你读"公众号文章 = 未经授权传播 |
| 腾讯微信开放平台协议 | 3.1 条：禁止抓取/爬取微信数据 | 任何"直读数据库"描述都违反此协议 |
