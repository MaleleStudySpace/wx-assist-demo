## 为什么要做这个项目？

微信是我们每天使用最久的 App，但它的信息管理能力还有不少提升空间：

- **群聊消息太多** — 几百条刷屏，你只想知道"今天群里决定了什么、谁在找我"
- **收藏夹是黑洞** — 收藏了 500 条内容，再也找不到了
- **公众号看不完** — 订阅了几十个，每篇都想看，但永远没时间
- **语音转瞬即逝** — 重要的语音消息听过就没了，无法搜索、无法回顾
- **想珍藏的拿不出来** — 聊天记录里的文字、图片、语音，总希望能妥善保存

**微信助手**就是为了解决这些日常痛点而生。

## ✨ 功能一览

### 🔔 关键词即时提醒

群聊里出现你关心的关键词，立刻推送微信通知——再也不怕错过重要消息。

<!-- 📷 此处插入：关键词提醒配置界面截图 — 展示预设关键词列表 + 剧本回放按钮 -->

- 预设常见关键词：`紧急BUG`、`线上问题`、`事故复盘`……
- 命中后自动推送微信（绑定后生效）
- 🎬 **剧本回放**：一键触发模拟群聊对话，体验从"消息发出"到"微信收到提醒"的全链路

---

### 📋 定时群摘要

不用翻聊天记录，每到固定时间自动生成"群简报"。

<!-- 📷 此处插入：定时摘要预览界面截图 — 展示 AI 生成的群摘要内容 -->

- 按话题分类，重点标注决定/行动项/未解决问题
- 支持群档案（群用途、关注点、忽略内容），摘要更精准
- 摘要可一键推送微信

### 📰 公众号定时摘要

几十个公众号，每天几十篇文章？让 AI 帮你读。

## 🛡️ 安全说明

| 保证 | 说明 |
|------|------|
| 🔒 只读展示 | Demo 仅展示模拟数据，不读写任何真实微信数据 |

## 📦 自部署 Demo

如果你想在自己的服务器上跑 Demo：

```bash
# 1. 克隆
git clone -b online-demo https://github.com/yourname/wx-assist-demo.git
cd wx-assist-demo

# 2. 构建前端
cd ui-src && npm install && npm run build && cd ..

# 3. 配置 AI 后端
cp .env.example data/.env
# 编辑 data/.env，填入 AI API 地址和 Key

# 4. 启动
python server.py
```

打开 **http://127.0.0.1:7328** 即可。

<details>
<summary>🔧 环境变量</summary>

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `DEMO_HOST` | `127.0.0.1` | 监听地址 |
| `DEMO_PORT` | `7328` | 监听端口 |
| `AI_PROVIDER_BASE_URL` | — | AI API 站点 URL |
| `AI_PROVIDER_API_KEY` | — | AI API Key |
| `AI_PROVIDER_TYPE` | `auto` | Provider 类型（auto/deepseek/claude） |
| `AI_PROVIDER_MODEL` | — | 模型 ID |
| `LOG_LEVEL` | `INFO` | 日志级别 |

</details>

<details>
<summary>🏗️ 技术栈</summary>

- **后端**：Python 3.13 + http.server + threading
- **前端**：React 19 + Vite 8 + Tailwind CSS v4 + Framer Motion + Phosphor Icons
- **AI**：DeepSeek / Claude / 任意 OpenAI 兼容 API
- **推送**：iLink Bot API（微信推送通道）

</details>
