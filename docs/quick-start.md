# 微信助手 Demo — 快速上手指南

> 5 分钟从零到体验全部功能

---

## 第一步：启动

```bash
# 确保在项目根目录
cd wx-assist-demo

# 启动后端
python server.py
```

看到 `wx-assist-demo server running at http://127.0.0.1:7328` 即成功。

打开浏览器访问 **http://127.0.0.1:7328**

---

## 第二步：配置 AI 后端

首次访问会进入引导页面。

1. 填写 AI 站点 URL（如 `https://api.deepseek.com`）
2. 填写 API Key（如 `sk-xxx`）
3. 点击「检测」→ 自动识别 Provider 类型和可用模型
4. 选择模型 → 保存

> 💡 没有配置 AI Key 也能体验 UI，但摘要和对话功能会提示"AI 后端未配置"。

---

## 第三步：体验核心功能

### ⚡ 关键词检测 — 1 分钟体验

1. 点击左侧 **「群聊助手」** Tab
2. 看到关键词提醒区域已有预设：技术交流群 + BUG/线上问题
3. 向下滚动找到 **「剧本回放」** 面板
4. 点击 **「▶ 开始回放」**
5. 观看消息逐条出现，3 条消息命中关键词时出现橙色标签

### 📰 公众号摘要 — 2 分钟体验

1. 点击左侧 **「公众号」** Tab
2. 看到 5 个公众号 + 2 个预设摘要分组（科技资讯 / 开发资源）
3. 点击「科技资讯」分组的 **「生成摘要」**
4. AI 根据该分组 7 篇文章生成摘要（tech 模板）
5. 摘要结果在卡片中展开查看

### 💬 AI 对话 — 1 分钟体验

1. 点击左侧 **「会话」** Tab
2. 选择任意群聊会话
3. 点击右上角 AI 图标开启对话
4. 输入问题，AI 基于群聊上下文回答

### 📊 首页总览

1. 点击左侧 **「首页」** Tab
2. 一览：服务状态、4 个健康指标、关键词提醒列表、4 个定时任务
3. 点击「环境检查」→ 诊断各组件状态

---

## 第四步：深度探索

### 自定义关键词

1. 群聊助手 → 关键词提醒 → 「+ 添加提醒群」
2. 从下拉选择群聊 → 输入关键词（回车添加）→ 保存
3. 再次运行剧本回放，观察新关键词是否命中

### 创建公众号摘要分组

1. 公众号 Tab → 「+ 新建分组」
2. 填写分组名 → 勾选公众号 → 选择模板和时间 → 保存
3. 点击「生成摘要」测试

### 绑定微信推送（可选）

1. 系统配置 → 微信推送
2. 点击「绑定」→ 用微信扫描二维码
3. 绑定后，关键词提醒和摘要可推送到微信
4. 在推送记录中查看推送历史

## 常见问题

### Q: 启动后浏览器打开是空白？

检查 `dist/` 目录是否有 `index.html` 和 `assets/`。如果没有，先构建前端：

```bash
node build.js
```

### Q: AI 摘要/对话提示"AI 后端未配置"？

需要在 `data/.env` 中配置 AI API Key，或在系统配置页面填写。

### Q: 关键词提醒刷新后消失？

助手配置会自动保存到 `data/assistant_config.json`。如果该文件被删除，会 fallback 到 `mock/assistant-config.json` 的预设数据。

### Q: 公众号列表为空？

Mock 数据有 5 个预设公众号。如果看不到，检查 `mock/oa-accounts.json` 是否存在。

### Q: 如何重置所有配置？

删除 `data/` 目录下的 `assistant_config.json`、`ilink_account.json`、`.env`，重启服务。
